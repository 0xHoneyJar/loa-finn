## Billing Lifecycle

<!-- provenance: CODE-FACTUAL -->
<!-- evidence: file=src/billing/state-machine.ts:157-500, file=src/hounfour/billing-finalize-client.ts:123 -->

**State machine**: IDLE → RESERVE_HELD → COMMITTED → FINALIZE_PENDING → FINALIZE_ACKED (or RELEASED, VOIDED, FINALIZE_FAILED)

### Reserve (`src/billing/state-machine.ts:181-221`)
1. Generate ULID `billingEntryId`
2. Create `BillingReservePayload`
3. WAL append (authoritative)
4. EventStore emit (fire-and-forget)
5. Redis update (derived cache)
6. Governed shadow (if `FINN_GOVERNED_BILLING=true`)
7. Return `BillingEntry` with `state=RESERVE_HELD`

### Commit (`src/billing/state-machine.ts:233-263`)
1. Validate transition RESERVE_HELD → COMMITTED
2. Create `BillingCommitPayload` with `actual_cost` (micro-USD)
3. WAL append
4. Redis update with `state=FINALIZE_PENDING`
5. Enqueue finalize to DLQ (async)

### Finalize (`src/hounfour/billing-finalize-client.ts:123`)
1. S2S JWT signed request to Arrakis `/api/internal/finalize`
2. On success: `finalized` or `idempotent` (409)
3. On failure: DLQ with exponential backoff (1m, 2m, 4m, 8m, 10m cap)
4. Terminal failures (401, 404, 422): no retry

### DLQ Replay
- Background timer every 5 minutes (`src/hounfour/billing-finalize-client.ts`)
- Claim lock prevents concurrent replays
- Max retries enforced per entry

## Goodhart Routing

<!-- provenance: CODE-FACTUAL -->
<!-- evidence: file=src/hounfour/goodhart/, file=src/hounfour/router.ts:50-73, file=src/index.ts:226-318 -->

**States**: disabled → shadow → enabled (also: init_failed)

| State | Behavior |
|-------|----------|
| disabled | Default model routing, no Goodhart influence |
| shadow | Goodhart evaluates but doesn't affect routing (observability only) |
| enabled | Goodhart actively selects providers based on quality signals |
| init_failed | Recovery scheduler runs with exponential backoff |

**Shared mutable holder** (`src/index.ts:230-236`): `goodhartRuntime` holder is read on every invoke request. Recovery scheduler updates it atomically when init succeeds. Changes propagate immediately to all in-flight requests without restart.

**Kill switch** (`src/hounfour/goodhart/kill-switch.ts`): Redis-backed, checked per request, <1s latency. Instant transition enabled → disabled.

**Exploration** (`src/hounfour/goodhart/exploration.ts`): Multi-armed bandit over provider pool with temporal decay (EMA-based) and quality signal feedback loop.

## Session Lifecycle

<!-- provenance: CODE-FACTUAL -->
<!-- evidence: file=src/agent/session.ts, file=src/gateway/ws.ts:45 -->

1. **Create**: `createLoaSession()` → load identity → init SessionManager → wrap tools with jail validation → create Pi SDK session
2. **Message**: WebSocket message → validate token → get/resume session → stream events through StreamBridge
3. **Compaction**: Auto-triggers when message count exceeds threshold — serialize state, write checksum, clear history
4. **Idle eviction**: SessionRouter evicts from memory after 30min idle. Disk state remains recoverable.
5. **Isolation**: Each session has independent message history and compaction state. No cross-session visibility.

## WAL Behavior

<!-- provenance: CODE-FACTUAL -->
<!-- evidence: file=src/persistence/wal.ts:46-100 -->

### Append (`src/persistence/wal.ts:69-115`)
1. Check disk pressure (hysteresis: LOW=100MB, HIGH=150MB)
2. Throw `DiskPressureError` if below threshold
3. Rotate segment if current ≥ 10MB
4. Generate ULID via `monotonicFactory()` (guaranteed ordering within process)
5. Compute SHA-256 checksum of JSON payload
6. Append JSONL entry: `{ id, timestamp, type, operation, path, data, checksum }`

### Segment Rotation
- New segment: `{walDir}/segment-{timestamp}-{ulid}`
- Atomic rename (no torn writes on crash)
- Reset `currentSegmentSize`

### Sync Cycle
- **R2 sync**: Every `config.syncIntervalMs` — upload new segments, persist checkpoint
- **Git sync**: Every `config.gitSyncIntervalMs` — tarball snapshot, commit + push
- **Pruning**: Hourly — delete segments with seq < min(R2 confirmed, git confirmed)

## Recovery Behavior

<!-- provenance: CODE-FACTUAL -->
<!-- evidence: file=src/persistence/recovery.ts:33-100 -->

**Cascade order**: template → R2 → git

| Source | Check | Restore | Timeout |
|--------|-------|---------|---------|
| Template | BEAUVOIR.md exists? | Write default if missing | 5s + 30s |
| R2 | Checkpoint available? | Download + extract | 5s + 30s |
| Git | WAL branch exists? | Clone/pull + extract | 5s + 30s |

**Overall deadline**: 120s. No hung source blocks boot indefinitely.

**Modes** (`src/persistence/recovery.ts:12`):
- **strict**: All sources must succeed; fail if any unavailable
- **degraded**: Proceed if ≥1 source succeeds
- **clean**: Skip recovery, start fresh

## Audit Trail

<!-- provenance: CODE-FACTUAL -->
<!-- evidence: file=src/hounfour/audit/buffered-audit-chain.ts, file=src/hounfour/audit/dynamo-audit.ts, file=src/hounfour/protocol-types.ts -->

### Dual-Format Hashing (v8.3.1)
- **Legacy**: `computeAuditEntryHash(entry, domainTag)` — content-only hash, no chain linkage
- **Chain-bound**: `computeChainBoundHash(entry, domainTag, prevHash)` — includes previous hash for tamper-evident ordering
- **Domain tags**: `buildDomainTag(storeName, version)` — natively sanitizes dots to hyphens (hounfour v8.3.1 PR #42)

### Buffered Append (`src/hounfour/audit/buffered-audit-chain.ts:81-89`)
1. Acquire sequential mutex (single-writer per process)
2. Try direct write to DynamoDB
3. On failure: buffer entry (in-memory, bounded)
4. **Critical actions** (routing_mode_change, settlement, admin_action): throw when buffer full
5. **Non-critical actions**: drop with warning when buffer full

### Chain Continuity
- Each entry: SHA-256 of (prev_hash + action + payload + timestamp)
- KMS signing for off-chain verification (S3 anchor)
- Sequence number gap detection reveals crashes

## Credit System

<!-- provenance: CODE-FACTUAL -->
<!-- evidence: file=src/credits/credit-persistence.ts:44-46 -->

**States**: ALLOCATED → UNLOCKED → RESERVED → CONSUMED (or EXPIRED)

**Conservation invariant** (`src/credits/credit-persistence.ts:44-46`):
```
allocated + unlocked + reserved + consumed + expired = initial_allocation
```
Enforced via SQL CHECK on every account update.

| Operation | Transition | Trigger |
|-----------|------------|---------|
| Purchase | → ALLOCATED | Account creation |
| Unlock | ALLOCATED → UNLOCKED | Proof-of-ownership signature |
| Reserve | UNLOCKED → RESERVED | Billing reserve |
| Consume | RESERVED → CONSUMED | Billing finalize |
| Expire | UNLOCKED → EXPIRED | TTL |

## Graceful Shutdown

<!-- provenance: CODE-FACTUAL -->
<!-- evidence: file=src/index.ts:776-857 -->

On SIGTERM/SIGINT:
1. Stop accepting new connections
2. Drain active WebSocket sessions (send `agent_end`)
3. Flush WAL to disk
4. Stop scheduler (cancel pending cron tasks)
5. Close Redis connection
6. Close PostgreSQL pool
7. Exit with code 0
